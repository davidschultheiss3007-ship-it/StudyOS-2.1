## Systematik der Performancemaße

Alle Performancemaße verzweigen in zwei Äste: **reine Renditemaße** und **risikoadjustierte Maße**. Letztere unterscheiden sich nur im verwendeten Risikomaß.

### Die zwei Zahlungsströme im Vergleich

<Columns count={2}>
<Card title="Marktsicht (TWR)" tone="teal" compact>
- +100 % im ersten Jahr
- −50 % im zweiten Jahr
- Höhe und Timing der Einzahlung **ausgeblendet**
</Card>
<Card title="Investorsicht (MWR)" tone="accent" compact>
- −100 € bei t=0, −300 € bei t=1
- +250 € Rückfluss bei t=2

</Card>

### Zeitgewichtete Rendite (TWR) — nur das Marktgeschehen

<Formula
  title="Zeitgewichtete Rendite (geometrisches Mittel der Periodenrenditen)"
  expr="r_{zeitgew.} = \sqrt{(1+r_1)(1+r_2)} - 1 = \sqrt{2 \cdot 0{,}5} - 1 = 0\,\%"
  legend={[
    { sym: 'r_1', desc: 'Periodenrendite Jahr 1 = +100 %' },
    { sym: 'r_2', desc: 'Periodenrendite Jahr 2 = −50 %' },
  ]}
/>

Die TWR ignoriert Höhe und Timing der Einzahlung und ergibt **0 %** — das Marktgeschehen war im Saldo neutral.

### Geldgewichtete Rendite (MWR) — Timing + Marktgeschehen

<Formula
  title="Geldgewichtete Rendite = interner Zinsfuß des Zahlungsstroms"
  expr="BW(r_g) = -100 - \frac{300}{(1+r_g)} + \frac{250}{(1+r_g)^2} = 0 \;\;\Rightarrow\;\; r_{geldgew.} = -32{,}06\,\%"
  legend={[
    { sym: 'r_g', desc: 'gesuchter interner Zinsfuß (IRR)' },
    { sym: '-100,\,-300', desc: 'Einzahlungen des Investors (t=0, t=1)' },
    { sym: '+250', desc: 'Rückfluss bei t=2' },
  ]}
/>

<Callout type="pitfall" title="Original-Tippfehler (Folie 9)">
Die Folie betitelt die geldgewichtete Rendite irrtümlich mit „TWR" — gemeint und korrekt ist **MWR** (Money-Weighted Rate of Return).
</Callout>

<CompareCard items={[
  { tag: 'MWR', title: 'Geldgewichtete Rendite', subtitle: 'z. B. Versicherung als Investor', accent: 'accent', points: [
    'interner Zinsfuß (IRR) eines Zahlungsstroms',
    'berücksichtigt **Zeitpunkte und Höhen** der Ein-/Auszahlungen',
    'zweckmäßig bei **bewusster** Beeinflussung der Kapitalflüsse',
    'Beispielergebnis: **−32,06 %**',
  ] },
  { tag: 'TWR', title: 'Zeitgewichtete Rendite', subtitle: 'z. B. Portfoliomanager / Fonds', accent: 'positive', points: [
    'geometrische Durchschnittsrendite der **Periodenrenditen**',
    'misst nur das **Marktgeschehen**, unabhängig vom Timing',
    'weltweit akzeptierter Standard (**GIPS**)',
    'Beispielergebnis: **0 %**',
  ] },
]} />

<Callout type="merksatz" title="Fazit: MWR vs. TWR als Timing-Test">
Die MWR drückt **zwei Effekte in einer Kennzahl** aus (Timing + Markt), die TWR **nur das Marktgeschehen**. Daher gilt: <Exam>MWR kleiner als TWR = ungünstiges Timing</Exam>, MWR größer als TWR = günstiges Timing der Inflows und Outflows.
</Callout>

## Sharpe-Ratio

<DefinitionBox term="Sharpe-Ratio" source="Folie 13 — William F. Sharpe, 1966">
Die **Risikoprämie pro Einheit des übernommenen Gesamtrisikos** (siehe CML). Sie verdichtet Rendite und Risiko zu einer Kennzahl.
</DefinitionBox>

<Formula
  title="Sharpe-Ratio"
  expr="SR_P = \frac{\mu_P - r_f}{\sigma_P}"
  legend={[
    { sym: '\mu_P - r_f', desc: 'Risikoprämie / Überschussrendite (Excess Return)' },
    { sym: '\sigma_P', desc: 'Gesamtrisiko = Volatilität (Standardabweichung)' },
  ]}
/>

Alle Größen müssen sich auf **denselben Zeitraum** beziehen (z. B. annualisiert).

## Treynor-Ratio

<DefinitionBox term="Treynor-Ratio" source="Folie 19 — Jack L. Treynor, 1965">
Die **Risikoprämie pro Einheit des übernommenen systematischen Risikos** (Beta). Sie misst — anders als Sharpe — die Überschussrendite nur am systematischen Risiko.
</DefinitionBox>

<Formula
  title="Treynor-Ratio"
  expr="TR_P = \frac{\mu_P - r_f}{\beta_P}"
  legend={[
    { sym: '\mu_P - r_f', desc: 'Risikoprämie / Überschussrendite' },
    { sym: '\beta_P', desc: 'systematisches Risiko = Betafaktor des Portfolios' },
  ]}
/>

<Callout type="pitfall" title="Original-Tippfehler (Folie 19)">
Die Folie schreibt im Definitionssatz „Sharpe-Ratio" statt **Treynor-Ratio** und beschriftet das systematische Risiko irrtümlich mit „σ_p" statt **β_p**.
</Callout>

<section id="jensen-alpha">

## Jensen-Alpha

<DefinitionBox term="Jensen-Alpha" source="Folie 24 — Michael C. Jensen, 1968">
**Absolutes Performancemaß** für den Renditeunterschied des Portfolios zur Benchmark unter Beachtung des Betas. Es misst die **Wertpapierselektionsfähigkeit** des Portfoliomanagers — den Teil der Rendite, der nicht mit der Benchmark korreliert.
</DefinitionBox>

<Formula
  title="Jensen-Alpha (Kurz- und ausführliche Form)"
  expr="\alpha_P = r_P - r_P^{CAPM} = \underbrace{(r_P - r_f)}_{\text{Gemessene Überrendite}} - \underbrace{(r_{BM} - r_f)\,\beta_P}_{\text{Erwartete Überrendite}}"
  legend={[
    { sym: 'r_P^{CAPM}', desc: 'nach CAPM erwartete Portfoliorendite' },
    { sym: '\beta_P', desc: 'Betafaktor des Portfolios' },
  ]}
/>

### Grafische Deutung: Abstand zur SML

Grafisch ist α der **vertikale Abstand** der tatsächlichen Rendite r_P zur CAPM-/SML-Linie:

- <Exam>Punkt über der Linie → α größer 0 (Outperformance)</Exam>
- Punkt unter der Linie → α kleiner 0 (Underperformance)
- Punkt auf der Linie → α = 0 (Rendite genau nach CAPM-Erwartung)

<DataChart
  type="scatter"
  title="Jensen-Alpha als Abstand zur Wertpapierlinie (SML, Folie 25)"
  source="Folie 25 — schematische Darstellung"
  xKey="beta"
  yKey="rendite"
  xLabel="Beta (β)"
  yLabel="Rendite"
  height={320}
  xDomain={[0, 1.6]}
  yDomain={[0, 16]}
  series={[
    { key: 'sml', name: 'Wertpapierlinie (SML)', color: 'var(--color-module-blue)', line: true, data: [
      { beta: 0, rendite: 3 }, { beta: 1.6, rendite: 14 },
    ] },
    { key: 'pf', name: 'tatsächliches r_P (α größer 0)', color: 'var(--color-module-teal)', data: [
      { beta: 1.2, rendite: 13 },
    ] },
  ]}
  markers={[
    { x: 1.2, y: 11.25, label: 'r_P(CAPM)', color: 'var(--color-module-orange)' },
  ]}
/>

Der orange Punkt ist die CAPM-erwartete Rendite bei β_P, der grüne die tatsächlich erzielte — ihr vertikaler Abstand ist α. Alternativ ist α der **Achsenabschnitt** der Regression von r_P auf r_BM:

<Formula
  title="Jensen-Alpha als Regressions-Achsenabschnitt"
  expr="r_P = \hat{\alpha}_P + \hat{\beta}_P \cdot r_{BM} + \varepsilon_P"
  legend={[
    { sym: '\\hat{\\alpha}_P', desc: 'Achsenabschnitt = Jensen-Alpha' },
    { sym: '\\hat{\\beta}_P', desc: 'Steigung = Betafaktor des Portfolios' },
    { sym: '\\varepsilon_P', desc: 'Residuum (unsystematischer Renditeanteil)' },
  ]}
/>

### Beispiel (Bruns/Meyer-Bullerdiek 2013)

<Columns count={2}>
<Card title="Portfolio A" tone="neutral" compact>
α_A = (4,5 % − 3 %) − (4 % − 3 %) · 1,2772 = **0,22 %**
</Card>
<Card title="Portfolio B" tone="accent" compact>
α_B = (4,3 % − 3 %) − (4 % − 3 %) · 0,9911 = **0,31 %**
</Card>
</Columns>

<Callout type="examfocus" title="Ergebnis">
Beide Portfolios haben ein **positives Jensen-Alpha** und schlagen damit die CAPM-Erwartung; <Exam>B weist mit 0,31 % das höhere Alpha auf</Exam> — bezogen auf das systematische Risiko performen A und B besser als die Benchmark.
</Callout>

## Bewertung der drei risikoadjustierten Maße

Die drei Maße ergänzen sich, sind aber unterschiedlich einsetzbar — die Wahl hängt von Risikomaß, Benchmark-Abhängigkeit und Ranking-Eignung ab.

<Callout type="pitfall" title="Jensen-Alpha taugt nicht zum Ranking">
Sharpe und Treynor erlauben ein Ranking, das Jensen-Alpha **nicht**: ein großes Alpha kann mit deutlich mehr Risiko erkauft sein. <Exam>Für den Investor ist relevant, wie viel Risiko zur Alpha-Erzielung zusätzlich in Kauf genommen wurde</Exam> — das bildet das Alpha allein nicht ab.
</Callout>
